使用说明
控件作用
  • 可限制上传文件类型
  • 可限制上传文件大小
  • 可限制上传文件数量
  • 上传图片有大小缩略图
  • 上传失败异常信息提示
  • 点击删除按钮删除图片
  • 可显示上传文件原始名
  • 可设置上传文件图标的数量
  • 可通过拍照上传照片
  • 点击图片进入图片编辑器
属性列表（上传控件指令：g-upload-group）
参数名 类型 事例值 说明
key String 1 上传控件策略，对应数据库表（fs_uploader_stragegy）的id号
ng-model Array<Object> [] 绑定文件信息，使页面初始化后显示已上传的文件信息
on-before-upload function(fileInfo, uploaderStrategy):Boolean 上传前事件，返回`false`时不上传，默认返回`true`
on-upload function(fileInfo, uploaderStrategy) 上传事件
on-before-delete function(fileInfo, uploaderStrategy):Boolean 删除前事件，返回`false`时不删除，默认返回`true`
on-delete function(fileInfo, uploaderStrategy) 删除事件
upload-params Object {rename: 'someNewFile'} 上传时的扩展参数，可在 `handler` 、 `listener` 中使用
delete-params Object {logic: true} 删除时的扩展参数，可在 `handler` 、 `listener` 中使用
initial-show-count Number 5 页面初始化时显示上传框的数量
show-filename Boolean false 上传后是否显示文件名，默认显示。true：显示文件名称；false：不显示文件名称
show-deleter Boolean `hover` 何时显示删除按钮，可为 `always`：始终；`never`：从不；`hover`：鼠标悬浮在图片时
keep-uploader-on-remove Boolean true 删除时是否删除上传框，默认删除。true：删除；false：不删除
disable-preview Boolean true 是否禁用图片预览，默认可预览。true：禁用预览；false：可预览
disable-upload Boolean true 在图片已经上传的前提条件下是否禁止文件再上传（例如：该功能用于头像显示）。true：禁止上传；false：可上传
g-snapshot <g-snapshot></g-snapshot> 拍照上传功能
简单用例
  • 该控件不是公共控件需要手动引入framework/uploader/UploadGroupModule上传控件模块
  • 在线拍照功能还需引入:framework/snapshot/GillionSnapshotModule模块
    • <script src="/bower_components/requirejs/require.js" data-main="/static/app.js"
     data-bootstrap-modules="['framework/uploader/UploadGroupModule','framework/snapshot/GillionSnapshotModule',
                              '/gschool/try/TestModule']">
</script>
  • 在html页面上使用上传控件
    • 上传文件：
<g-upload-group
            initial-show-count="1"
            upload-params="uploadParams"
            delete-params="deleteParams"
            show-deleter="hover"
            ng-model="fileInfos"
            on-before-upload="beforeUpload(fileInfo, uploaderStrategy)"
            on-upload="upload(fileInfo, uploaderStrategy)"
            on-before-delete="beforeDelete(fileInfo, uploaderStrategy)"
            >
</g-upload-group>
