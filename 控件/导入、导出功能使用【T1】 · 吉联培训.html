EXPORT_CONFIG:

列名 字段说明
id 主键，对应前台的key值
template EXCEL模板(二进制数据存储)
async 是否异步打印(0：否；1：是)
dictionary_url 获取数据字典controller的URL地址，如：/ec-demo/dictionary
file_name 定义导出EXCEL的文件名

class_name 打印数据的实体类名(全限定名)
source_url 获取导出的数据源地址，即controller的映射地址
column_setting 分组表格导出，不使用excel模板时，在此配置column
statistics_url 分组表格导出时，获取最后一行统计值的Controller的URL地址
sort_type 分组表格导出方式，共3种方式.0,1,2
1.2.2. EXCEL模板说明

使用方法：#!属性名称

导出格式 模板 说明
文本格式 #!属性名称
数字格式 #!属性名称{"type":"number","format":"######.00"} type指number类型，format的#代表整数保留位数，小数点后面0的个数代表小数保留位数，若位数不够补零
日期格式 #!属性名称{"type":"date","format":"yyyy-MM-dd"} type指date类型，format指日期格式，默认format格式为"yyyy-MM-dd hh:mm:ss"
布尔类型 #!属性名称{"values":{"true":"对应值1","false":"对应值2"}} values指导出布尔类型对应值
临时数据字典(小量的) #!属性名称{"values":{"x0":"对应值1","x1":"对应值2","x3":省略...}} 导出数据时需对相应字段进行转换，当转换数量较少时可使用临时数据字典进行配置转换
大量数据字典 #!属性名称{"dictionary":"dictName"} 导出数据时需对相应字段进行转换，当转换数量较多时可使用大量数据字典进行配置，具体实例如下所示

其中，籍贯字段的配置依据如下所示（即大量数据字典配置）

......

@Controller
@RequestMapping("/dictionary")
public class DictionaryController {
    @Autowired
    private AreaRepositoryImpl areaRepositoryImpl;

    @RequestMapping(method = RequestMethod.GET)
    @ResponseBody
    public Map<String,Object> returnDictionary(@RequestParam(required = false) String ...params){
        Map<String,Object> mapData = new HashMap<String,Object>();
        HashMap<String,Object> dictMap=new HashMap<String,Object>();

        Iterable<Area> iterables = areaRepositoryImpl.findAll();
        for(Area iterable:iterables){
            dictMap.put( iterable.getCode(), iterable.getName());
        }
        mapData.put("area",dictMap);
        return ResultUtils.getSuccessResultData(mapData);
    }
}

  • 此时导出数据字典使用模板：#!place{"dictionary":"area"}，其中area为DictionaryController中mapData集合的key值 excel模板还可配置导出时的列样式功能。操作简单，只需要设置excel导出表头的单元格格式即可。例如：

  • 设置某列导出后字体大小

2.7.1 Excel导入数据库配置

EXCEL导入配置信息可在数据库中进行配置维护，也可以直接使用代码将配置信息写入，下面介绍使用数据库配置方式来进行配置。

import_config:

列名 字段说明
id 主键，对应前台的key值
template EXCEL模板（二进制数据存储），具体配置方法见5.1.2.1
class_name 读取excel转化成的实体对象的类名(全限定名)
dictionary_url 获取数据字典controller的URL地址
deal_service 结果处理类
method_name 结果处理类的方法名
allow_parameter 类处理方法是否允许带参数，0代表不允许，1代表允许。允许：方法为dealExcelList(List excelData,Map param) ；不允许：dealExcelList(List excelData)
validate_head 校验导入的excel文件表头与模板表头是否一一对应。1：校验；0：不校验。当校验不通过时弹出提示窗：导入的EXCEL文件表头设置错误
async 是否异步导入(0：否；1：是)，具体介绍见1.3
primary_columns 主表列属性
define_column 主从表列属性
children_table 从表属性名定义
validate_method 行校验方法。在导入的过程中可以对传入的EXCEL每一行进行校验。只需要在数据库(import_config)中，配置validate_method属性。如果校验的过程中要退出，直接抛异常。也可以对每一行返回校验信息。校验信息会在结果处理方法的param中。
